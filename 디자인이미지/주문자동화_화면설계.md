# WEGO 2.0 · 주문자동화 화면설계 (v0.1)

> 본 문서는 “화면에 들어가야 할 것(content & component)”을 정의한 화면설계서입니다.
> 시각 GUI는 후속 디자인 단계에서 정합화하며, 본 문서는 정보 구조 / 컨텐츠 / 액션 기준으로 작성됩니다.

- 대상 : 성림에너지 Pilot (원곡 · 평택 · 안산)
- 사용자 : 충전소 담당자(소장) / 운영관리자 / 기사
- 요구사항 매핑 : SKBS_AMO_RQ_01 ~ 09
- 작성일 : 2026-04-28

## S01. 메인 대시보드 (홈)

**목적** : 출근 직후 “오늘 무엇을 결정해야 하는가”를 한 페이지에 노출.
**진입 경로** : 로그인 직후 / GNB “대시보드”

### 화면 구성 요소

#### 1) 글로벌 헤더 (모든 화면 공통)
- 로고 (WEGO 2.0)
- GNB : 대시보드 / 주문관리 / 수요처관리 / 모니터링 / 분석·KPI / 설정
- 충전소 선택 드롭다운 (원곡 / 평택 / 안산 / 전체)
- 현재 사용자 + 권한 뱃지 (소장 / 관리자 / 기사)
- 알림 뱃지 (안 읽은 건수)

#### 2) 오늘의 KPI 카드 (4개)
| 카드 | 표시 내용 | 비고 |
|---|---|---|
| 추천 주문 | 건수 / 어제 대비 증감 | 클릭 → S02 |
| 긴급(잔량 < 20%) | 건수 / 즉시 처리 안내 | 클릭 → S02 (필터: 긴급) |
| 예상 출고량 | 톤 / 차량 대수 | 클릭 → 배차 화면 |
| 자동화 커버리지(%) | 수기 개입 없는 비중 | 클릭 → S10 |

#### 3) 추천 주문 Top N (요약 리스트)
- 컬럼 : 우선순위(P0/P1/P2) · 수요처명 · 잔량(%) · 예상 소진일시 · 권장량 · 상태
- 행 액션 : [확정] [상세] [보류]
- 상단 액션 : [선택 일괄 확정] [모델 재실행] [엑셀 내보내기]

#### 4) 알림 / 예외 카드
- 항목당 표시 : 심각도 아이콘 · 제목 · 수요처 · 발생시각 · [확인][조치] 버튼
- 분류 : 센서 이상 / 사용상태 변경(휴업) / 배차부하 초과 / 모델 신뢰도 저하

#### 5) 충전소 S/B 재고 현황
- 탱크별 게이지 (S/B1, S/B2, … 충전소 본탱크) — 잔량(%) + kg
- 표시 정보 : 오늘 출고 예정량 · 충전 가용량 · 긴급 라인 가동 여부

#### 6) 오늘 배차 부하 미니차트
- X : 시간대(0~24h)
- Y : 예상 출고량 vs 가용 CAPA(점선)
- 초과 시간대는 강조 + "후행 조정 권장" 메시지

### 데이터 / 연동
- WEGO 주문, 스마트미터 잔량, 차량 운행 스케줄, 모델 추천 결과
- 자동 새로고침 5분 (수동 새로고침 버튼 제공)

### 요구사항 매핑
RQ_03 / RQ_04 / RQ_06 / RQ_07 / RQ_08 / RQ_09

## S02. 추천 주문 리스트

**목적** : AI가 산출한 주문 후보를 검토·확정.
**진입** : GNB → 주문관리 → 추천 주문 / S01 카드 클릭

### 화면 구성 요소

#### 1) 페이지 헤더
- 타이틀 + 기준일시 (예 “2026-04-28 08:00 기준”)
- 우측 액션 : [모델 재실행] [엑셀 내보내기] [선택 일괄 확정 (n)]

#### 2) 탭 (Status 분류)
- 전체 (n) / 긴급 (n) / 주의 (n) / 정상 (n) / 보류·예외 (n) / 확정완료 (n)

#### 3) 필터 바
- 충전소 / 업종 / 군집(C-1~C-4 / 보조) / 납품일(D+0~D+7) / SM 상태 / 우선순위 / 모델 신뢰도(슬라이더)
- 검색어 : 수요처명 / 코드 / 주소
- 저장된 필터 프리셋 (잔량<20%, D+1, 고수요분할, 신규/SM미설치)

#### 4) 추천 주문 테이블
| 컬럼 | 설명 | 표시 형식 |
|---|---|---|
| 선택 | 다중선택 체크박스 | — |
| 우선순위 | P0(긴급)·P1(주의)·P2(정상) | 컬러 뱃지 |
| 수요처 | 명 + 코드(SM-xxx) | 텍스트 + 보조 |
| 업종/군집 | 산업·상업·농어업 / C-n | 텍스트 |
| 잔량 | 현재 % + 임계값 | 게이지 + 마커 |
| 예상 소진 | 일시 (D+n hh:mm) | 텍스트 |
| 권장량 | kg + 분할 여부(1/2 등) | 텍스트 |
| 차량 | 추천 차량 호차 | 텍스트 |
| 모델 신뢰도 | % | 컬러 뱃지 |
| SM 상태 | 정상/샘플링↑/누락/미설치/오류 | pill |
| 액션 | [확정][상세][보류][수기수정] | 버튼 |

#### 5) 동적 시점 조정 안내(인라인 알림)
- D+n 가용 CAPA 초과 시 : “P2 일부를 D+n+1로 후행 조정 권장” + [자동 재배치 적용] 버튼

#### 6) Bulk 동작
- 선택 후 : 일괄 확정 / 일괄 보류 / 일괄 차량 변경 / 일괄 사유 입력

### 정렬 / 페이지네이션
- 기본 정렬 : 우선순위 ▶ 예상 소진시각 오름차순
- 페이지 단위 : 25 / 50 / 100

### 요구사항 매핑
RQ_01 / RQ_02 / RQ_03 / RQ_04 / RQ_05 / RQ_06

## S03. 추천 주문 상세 (모달 / 팝업)

**목적** : 단일 주문 후보를 검토·확정.
**진입** : S02 행 → [상세] 또는 행 클릭

### 화면 구성 요소

#### 1) 헤더 영역
- 수요처명 + 코드 (SM-Bxx-xxx)
- 분류 뱃지 : 업종 · 군집 · 체적/중량 · 차량배차 호차
- 상태 뱃지 : P0/P1/P2 · SM 상태 · 모델 신뢰도

#### 2) 잔량 & 예측 차트
- X : 시간 (과거 14일 + 예측 D+0~D+3)
- Y : 잔량(%) + 절대량(kg)
- 라인 : 실측(실선) / 예측(점선) / 신뢰구간(밴드)
- 마커 : 이전 납품 이력 / 현재 잔량 / 안전재고 임계선

#### 3) 권장 주문 산출 표
| 항목 | 값 |
|---|---|
| 현재 잔량 | %, kg |
| 리드타임 (D+n hh:mm) | — |
| 리드타임 내 예상 소모 | kg |
| 안전재고 동적 임계 | %, kg |
| 탱크 용량 / 여유 | kg |
| 차량 적재 한도 | kg |
| 권장 충전량 | kg (강조) |
| 분할 권고 | 1/2, 다음 회차 일시 |

#### 4) 결정 근거 패널 (AI Explainability)
- 군집 평균 일소비량 / Daily Max
- 영향 큰 외생변수 (기온·가동률·요일성)
- 파생변수 요약 (Rolling max 1h, Time-lag 7h …)
- 안전재고 정책 (Hard Fail 15% 방지, 동적 임계 18% …)
- 최근 30일 적중률
- 신뢰도 산출 근거 짧은 자연어 설명

#### 5) 최종 입력 폼
- 납품일 (datepicker, 기본 추천일)
- 주문량 (수량 입력 + [최소량] [권장량] 빠른버튼)
- 차량 (드롭다운, 적재 잔량 표시)
- 주문 사유 (선택, 기본 “AI 추천 · 자동”)
- 메모 (자유 입력)

#### 6) 푸터 액션
- [사유 추가] [⏸ 보류] [수기 수정] [✓ 주문 확정]
- 보류 / 수기 수정 클릭 시 → S05 예외 처리 모달 호출

### 데이터 갱신
- 모달 오픈 시점에 최신 잔량·예측을 재요청

### 요구사항 매핑
RQ_03 / RQ_04 / RQ_05 / RQ_07 / RQ_08

## S04. 일괄 확정 (모달 / 팝업)

**목적** : 다건 추천을 한 번에 확정 — 시스템 검증 후 처리.
**진입** : S01 / S02 [선택 일괄 확정]

### 화면 구성 요소

#### 1) 요약 카드
- 선택 건수 · 합계 권장량 · 사용 차량 수 · 예상 소요시간
- 가용 CAPA 대비 사용률 게이지

#### 2) 검증 결과 테이블
- 컬럼 : 수요처 · 권장량 · 차량 · 검증결과 · 메모
- 검증결과 라벨
  - OK : 임계 충족 · 차량 적재 가능
  - WARN : CAPA 90%↑ · 신뢰도 < 70%
  - FAIL : 차량 부족 · 동일 시간대 충돌 · 휴업 추정
- FAIL 행은 자동 제외 (체크 해제) + 사유 표시

#### 3) 후행 조정 옵션
- “WARN/FAIL 일부를 D+n+1로 자동 재배치하시겠습니까?” [예 / 아니오]
- 배치 결과 미리보기 (Before/After 차이만 표시)

#### 4) 푸터 액션
- [닫기] [선택만 확정] [재배치 후 확정 (권장)]
- 처리 진행률 표시 (n/N)

### 처리 결과
- 성공/실패 카운트 + 실패 건 사유 리스트
- 실패 건은 S02 “보류·예외” 탭으로 이동

### 요구사항 매핑
RQ_05 / RQ_06

## S05. 예외 처리 (모달 / 팝업)

**목적** : 주문 후보를 “주문 외 상태”로 전환 또는 수기 보정.
**진입** : 모든 화면의 [보류] / [예외] 버튼

### 사유 카테고리 (6종, 라디오)
| 사유 | 설명 | 후속 처리 |
|---|---|---|
| 다시 알림 (보류) | 일시 보류 후 재추천 | +N일 후 자동 재추천 (기본 3일) |
| 거래 중단 | 휴업·공사·행사 | 종료일까지 추천 제외 |
| 비활성 | 시즌 미사용 | 활성화 전까지 제외 |
| 누락 / Data 오류 | 잔량·납품 데이터 이상 | 수기 보정 + 모델 학습 제외 |
| 수기 접수 | 전화·팩스 등 외부 접수 | 수기 주문으로 신규 등록 |
| 거래 종료 | 영구 제외 | 수요처 비활성화 |

### 입력 항목
- 사유 카테고리 (필수)
- 적용 기간 (해당 사유에 한함)
- 메모 (필수, 다시알림은 선택)
- 첨부 (사진·문서, 선택)

### 푸터 액션
- [취소] [저장 및 적용]

### 요구사항 매핑
RQ_07 / RQ_08

## S06. 수요처 리스트

**목적** : 전체 수요처 검색·필터·일괄 정책 적용.

### 화면 구성 요소
- 검색바 : 수요처명 / 코드 / 주소 / 사업자번호
- 필터 : 충전소 · 업종 · 군집 · SM 설치여부 · 거래상태 · 안전재고 정책 적용 여부
- 테이블 컬럼 :
  - 수요처명 / 코드
  - 업종 / 군집
  - 탱크 용량 / 잔량(%)
  - 평균 사용량 (kg/일)
  - 최근 주문일
  - SM 상태
  - 거래 상태 (정상/휴업/종료)
  - 안전재고 정책 (기본/Custom)
- 행 클릭 → S07 수요처 상세
- Bulk : 정책 일괄 적용 / 군집 재할당 / 비활성 처리

## S07. 수요처 상세

**목적** : 개별 수요처 프로파일·예측·정책 통합 관리.

### 탭 구성
1. 개요
2. 잔량 / 예측
3. 주문 이력
4. 스마트미터
5. 안전재고 정책
6. 알림

### 1) 개요 탭
- 마스터 정보 : 명·코드·주소·담당자·연락처·사업자
- 군집 정보 : 군집명 · 군집 중심과의 유사도 · 군집 변경 이력
- 거래 상태 : 정상/휴업/종료, 변경 이력
- 핵심 지표 카드
  - 평균 사용량 (kg/일)
  - 소비속도 (kg/h)
  - 주문 주기 (일)
  - 충전 비중 (%)
  - 관측 일수
  - 고소비 구간 (시간대)

### 2) 잔량 / 예측 탭
- 잔량 시계열 차트 (실측 / D+0~D+3 예측 / 신뢰구간 / 안전재고선 / 납품 마커)
- 파생 변수 패널
  - Rolling Max 1h / Mean 1h / Range
  - Time-lag 1h / 3h / 7h
  - 외생변수 : 기온·강수·습도(7일 평균)
- 예측 정확도 (최근 30일) — MAPE / 적중률(±10%)

### 3) 주문 이력 탭
- 최근 N주(기본 12주) 주문 캘린더 + 막대그래프
- 표 : 주문일 / 납품일 / 주문량 / 실납품량 / 적중여부 / 사유

### 4) 스마트미터 탭
- 단말 정보 : 모델·설치일·배터리·통신상태
- 수신 주기 (현재) / 자동 조정 이력
- 최근 이상 이력 표

### 5) 안전재고 정책 탭
- 기본/Custom 선택
- 입력 항목
  - Hard Fail 임계 (% 또는 kg, 기본 15%)
  - 리드타임 (D+1 ~ D+2)
  - Daily Max 사용량 (kg)
  - 리드타임 변동성 σ (일)
  - 동적 임계 (자동 산출, 수정 가능)
- 시뮬레이션 : 임계값 슬라이더 → 월 주문 발생 빈도 변화 미리보기
- 변경 이력

### 6) 알림 탭
- 알림 채널 (이메일/SMS/푸시) on-off
- 임계 도달 / 센서 이상 / 사용상태 변경 별 채널 매핑
- 담당자 다중 지정

## S08. 스마트미터 모니터링

**목적** : 센서 이상 / 사용상태 자동 판단 / 수신 주기 최적화.

### 1) KPI 카드
- 전체 SM 수 / 정상 수 / 이상 감지 수 / 샘플링 자동 조정 수

### 2) 이상 감지 리스트
- 컬럼 : 심각도(High/Mid/Low) / 수요처 / 이상 유형 / 발생시각 / 현재 상태 / 조치
- 이상 유형
  - 납품 중인데 잔량 무변동
  - 납품 없는데 잔량 급증
  - 수신 끊김 (n시간 이상)
  - 3일 연속 사용 0kg (휴업 추정)
  - 이상 과다 / 이상 저하 소비
  - 데이터 정합성 오류
- 행 액션 : [현장 점검] [통신사 확인] [상태 변경(휴업)] [자동복구 로그]

### 3) 사용 상태 자동 판단
- 휴업/공장 중단 자동 감지 → 주문 생성 제외
- 이상 저하 / 이상 과다 소비 패턴 분류 + 알림
- 공장 운영 캘린더 (명절·법정공휴일) 연계

### 4) 수신 주기 최적화 정책
- 표 : 탱크 규격 / 고소비 구간 / 권장 샘플링
- 이상 발생 시 자동 ↑ 1단계 + 24h 모니터링
- 배터리 효율 알림 (80% 이하 → 교체 권고)

## 부록 A. 공통 컴포넌트 정의

### A1. 잔량 게이지
- 표시 : 현재 % + 절대량(kg, 보조)

- 안전재고 임계값 마커(검정 라인) 위에 오버레이

- P2 (Blue, “정상”) — 잔량 30%↑

### A3. 신뢰도 뱃지
- ≥85% Green / 70~84% Yellow / <70% Red
- Tooltip : “최근 30일 적중률 + 군집 평균 신뢰도”

### A4. SM 상태 pill
- 정상 / 샘플링↑ / 통신이상 / 누락 / 미설치 / 오류

### A5. 사유 코드 (공통)
