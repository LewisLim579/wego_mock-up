# 주문자동화 비즈니스 프로세스 작도서 (BPMN-ready, v0.1)

> 본 문서는 비즈니스 프로세스 다이어그램(BPMN/Visio/draw.io/Figma 등)으로 **그대로 그릴 수 있도록** 작성되었다.
> 풀(Pool)·레인(Lane)·태스크(Task)·게이트웨이(Gateway)·이벤트(Event)·플로우(Flow)·메시지(Message) 단위로 구조화한다.
> 상세 정의서는 `주문자동화_프로세스_정의서.md` 참조.

- 작도 권장 도구 : draw.io / Lucidchart / Visio / Figma + BPMN plugin
- 표기 표준 : BPMN 2.0 기본 도형 사용

## 0. 표기 규칙

### 0-1. 도형
| 도형 | 의미 | 예 |
|---|---|---|
| 둥근 사각형 | 태스크(활동) | "추천 검토" |
| 마름모 (◇) | 게이트웨이(분기) | "확정 / 보류 / 수기" |
| 원 (얇은선) | 시작 이벤트 | "샘플링 주기 도달" |
| 원 (굵은선) | 종료 이벤트 | "정산 완료" |
| 원 (이중선) | 중간 이벤트 | "이상 징후 수신" |
| 봉투 | 메시지 이벤트 | API 송수신 |
| 시계 | 타이머 이벤트 | "마감 주기" |

### 0-2. 플로우(연결선)
| 선 | 의미 |
|---|---|
| 실선 화살표 | 시퀀스 플로우 (같은 풀 내) |
| 점선 화살표 | 메시지 플로우 (풀 간) |
| 굵은 실선 | 핵심 경로(Happy Path) |
| 노란 음영 라벨 | 비즈니스 룰 적용 지점 |

### 0-3. 식별자(ID) 명명규칙
- 태스크: `T###`
- 게이트웨이: `G###`
- 이벤트: `E###`
- 메시지: `M###`
- 데이터 객체: `D###`

## 8. 서브 프로세스 (별도 BP로 분리 권장)

작도 시 서브프로세스 박스(접힌 사각형)로 분리하는 것을 권장한다.

### SP-1. 보류 수요처 관리 서브프로세스

```
[보류 등록(T260)] ──► (보류 풀 적재) ──► ◇ G110 만료?
                                          ├── 도래 ──► (자동 복귀) ──► T210
                                          └── 미도래 ──► (모니터링) ─►(연장/종료/사유 변경) ─► T290
```

| 사유 | 만료 룰 |
|---|---|
| 다시 알림 | +3일 |
| 거래 중단 | 종료일 도래 |
| 비활성 | 시즌 종료 |
| 누락 | 24시간 |
| 수기 접수 | 즉시 |
| 거래 종료 | 무기한 |

### SP-2. 이상 징후 처리 서브프로세스

```
[T600 이상 표시] ──► ◇ G700 심각도
                      ├── High ──► T620 샘플링↑ + 알림 ──► T610 현장점검
                      ├── Mid  ──► T620 샘플링↑ + 24h 모니터
                      └── Low  ──► (로그) ── 자동 복구
```

### SP-3. 정산 서브프로세스

```
[T500 마감] ──► T510 수송비 산정 ──► ◇ G500 분쟁?
                                       ├── 없음 ──► T530 외부 회계 연계 ──► [E007]
```
